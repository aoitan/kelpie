# SKILL: work-breakdown

## 目的
設計を具体的かつ並列可能なタスクに分解し、実装計画を立てる。

## この工程で重視すること
- 詳細度
- 依存性の可視化
- 実行可能性

## 手順
1. 設計書から必要な作業項目を抽出する
2. 作業項目間の依存関係をマッピングする
3. 1タスクが大きすぎないよう調整する（1 turn で終わる粒度が理想）
4. 実装順序を決定する

## 成果物チェック
- すべての作業が網羅されているか
- 依存関係が明確か
- 実装順序に無理がないか
