# SKILL: implementation

## 目的
計画に沿って、差分を小さく保ちながら実装する。

## この工程で重視すること
- 計画との整合
- 差分の小ささ
- 変更理由の明示
- 後工程のレビュー容易性

## 手順
1. 計画から今回着手範囲を確認する
2. 必要最小限の変更を入れる
3. 変更理由と差分を記録する
4. ローカル確認を行う
5. 次工程に見てほしい点を残す

## 避けること
- ついで修正の混入
- 大きすぎる差分
- 計画差分の無記録

## 成果物チェック
- 何を変えたか明確か
- 変更ファイルが列挙されているか
- 未対応事項が明示されているか
- ローカル確認内容があるか
