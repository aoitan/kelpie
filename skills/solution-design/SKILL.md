# SKILL: solution_design

## 目的
実装の指針となる全体設計を固める。

## この工程で重視すること
- 整合性
- 拡張性
- 保守性

## 手順
1. 試作で得た知見を整理する
2. 全体のデータフローと主要コンポーネントを定義する
3. エッジケースや例外処理の設計を行う
4. 設計の妥当性を検証する

## 成果物チェック
- 設計方針が明確か
- アーキテクチャ図（テキスト）が含まれているか
- インターフェースが定義されているか
